# Frameworks

## A. Intake Framework: S.O.F.T.Risk

Use before giving scripts.

- **S — Scene**: 发生了什么？在哪个渠道？面对面、电话、微信、邮件、会议？
- **O — Other person**: 对方是谁？上司、下属、同事、客户、伴侣、家人？权力关系如何？
- **F — Feeling**: 用户情绪强度0-10？主要情绪是什么？身体反应如何？
- **T — Target**: 用户想达成什么？解释、拒绝、修复、说服、设边界、投诉、离开？
- **Risk**: 有无暴力、自伤、威胁、骚扰、法律、HR、客户升级、职业后果？

If emotion intensity is 8-10, advise pause and regulation before crafting final message.

## B. Emotion Regulation: STOP-6

Use when user is angry, panicked, humiliated, or ready to send a harsh message.

1. **Stop**: 暂停发送、暂停争辩。
2. **Take breath**: 做6轮慢呼吸，延长呼气。
3. **Observe**: 命名情绪：“我现在是愤怒/委屈/害怕。”
4. **Pinpoint need**: 识别需求：“我需要尊重/公平/安全/清晰。”
5. **Choose goal**: 选择目标：“我要解决问题，不是发泄伤人。”
6. **Proceed**: 用低冲突句式行动。

Quick line:

“我现在情绪有点高，担心说得不准确。我们暂停10分钟/下午再继续，我会带着具体方案回来。”

## C. Facts-Feelings-Needs-Requests: FFNR

For feedback, refusal, conflict repair, and boundary setting.

- **Facts**: 可观察事实，避免“总是”“故意”“不尊重”等判断。
- **Feelings**: 自己的感受，避免指责对方导致。
- **Needs**: 背后的需求或原则。
- **Requests**: 具体、可执行、可接受或可拒绝的请求。

Template:

“关于【事实】，我感到【感受】，因为我需要【需求/原则】。我想请求【具体行动】，你看是否可行？”

## D. Reflect-Validate-Ask-Share: RVAS

For angry customers, criticized users, and relationship conflict.

1. **Reflect**: 复述内容：“你遇到的是……”
2. **Validate**: 承认情绪合理性：“换成我也会着急/失望。”
3. **Ask**: 问关键问题：“你最希望先解决哪一点？”
4. **Share**: 表达方案或限制：“我能做的是……不能承诺的是……”

## E. Ethical Influence: TRUST

Use for persuasion and leadership influence.

- **T — Trust**: 先建立可信度：事实、承诺、透明。
- **R — Reciprocity**: 说明互惠：对对方有什么价值。
- **U — Understanding**: 先理解对方目标、顾虑、压力。
- **S — Shared criteria**: 使用共同标准：成本、风险、客户价值、团队目标。
- **T — Trial next step**: 提出小步试行，而非强迫全盘接受。

Template:

“我理解你担心【顾虑】。如果用【共同标准】看，我的方案能带来【价值】，风险是【风险】。我建议先用【小范围试点】验证，结果不好我们再调整。”

## F. Boundary Framework: Kind-Clear-Firm

For refusal, overload, disrespect, or recurring requests.

1. **Kind**: 承认对方需求或关系。
2. **Clear**: 明确自己不能做什么。
3. **Firm**: 不过度解释，不反复道歉。
4. **Alternative**: 给可行替代方案。

Template:

“我理解这件事对你很急。今天我不能接这个任务，因为我必须在下班前完成A。可选方案是：我明天上午帮你看30分钟，或者你先找B确认第一版。”

## G. Feedback Framework: SBI-Growth

For managers, peers, and relationship feedback.

- **Situation**: 哪个场景？
- **Behavior**: 对方做了什么可观察行为？
- **Impact**: 造成什么影响？
- **Growth request**: 希望下次怎样做？

Template:

“在【场景】里，我注意到【行为】。它带来的影响是【影响】。下次我希望我们可以【具体改进】。”

## H. Apology and Repair: A.C.T.

Use when user made a mistake.

- **Acknowledge**: 承认具体影响，不辩解。
- **Care**: 表达对对方感受和损失的重视。
- **Take action**: 说明补救和预防。

Template:

“昨天我在会议上打断你两次，这影响了你完整表达。我为这一点道歉。下次你汇报时我会先记录问题，等你讲完再问；如果我又打断，请你直接提醒我。”

## I. Role-play Loop

When user asks to practice:

1. Ask user to choose role: “你演自己，我演对方” or “我先示范”。
2. Set difficulty: gentle, realistic, difficult.
3. Simulate one turn at a time.
4. After each user reply, score on: calmness, empathy, clarity, boundary, next step.
5. Offer a better version and let user retry.

## J. Post-conversation Review Scorecard

Score 1-5:

- 自我觉察：是否识别情绪和需求？
- 情绪调节：是否避免冲动攻击？
- 同理倾听：是否复述对方核心关切？
- 表达清晰：是否说清事实、感受、需求、请求？
- 边界感：是否明确可接受和不可接受？
- 非操控影响力：是否基于共同利益和信任？
- 后续行动：是否有下一步和记录？

Then provide:

- 做得好的2点。
- 可改进的2点。
- 下一次可直接使用的一句话。
